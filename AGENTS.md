# AGENTS.md (Baton Studio)

This repository is an **end-to-end product build**. Follow these rules:

## Build goals
- Implement Baton Studio according to `docs/PRD.md`, `docs/UX_SPEC.md`, and `docs/ARCHITECTURE.md`.
- Ship a **non-boring**, premium UI. The UI is a core deliverable.
- Provide a deterministic demo mode (no API keys) and a real MCP integration mode.

## Non-negotiables
- `make dev` starts backend + frontend reliably.
- `make check` runs backend format/lint/typecheck/tests, frontend lint/typecheck, and MCP smoke tests.
- `make e2e` self-starts backend + frontend, runs Playwright, and refreshes screenshots under `assets/ui/`.
- `make demo` runs a demo script and produces `dist/demo_pack.zip` plus a production frontend build.

## Tech constraints
- Local-first by default:
  - SQLite for persistence.
  - No cloud required for demo mode.
- Use WebSocket for realtime updates to the UI.
- Maintain an **append-only event log** (for replay and provenance).

## UI quality bar
- Do NOT ship default template UI.
- Must include:
  - Mission Control dashboard
  - World Model inspector (typed objects, diffs)
  - Causal Graph view (interactive)
  - Baton + Energy HUD with animation
  - Onboarding that loads a demo mission in 1 click

## Output expectations
- A world-class README and docs that match the implementation.
- Include diagrams under `assets/diagrams/`.
- Include sample screenshots under `assets/ui/` (generated by Playwright if possible).

## Commands
Prefer:
- `make dev`
- `make check`
- `make e2e`
- `make demo`

Do not introduce complex infrastructure unless it materially improves outcomes.
